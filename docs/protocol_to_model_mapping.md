# Protocol-to-model mapping

| Manuscript section | Concrete protocol element | HLPSL representation | Security meaning preserved | Abstraction or limitation |
|---|---|---|---|---|
| Local M1 | `V -> ATAj : {IDV, request, t0}` | `local_v` transition 1 sends `IDV.req.T0'`; `local_ataj` transition 1 receives `IDV.req.T0'` | Starts the local-authentication session with a fresh symbolic timestamp | `T0'` is `new()`, not a numeric clock value |
| Local M2 | `ATAj` generates `Ci`, `Ns`, `t1` and sends `{Ci, Ms, Auths, t1}` | `local_ataj` transition 1 creates `Ci'`, `Ns'`, `T1'`; `Ms' := Mask(Ns', PUF(PUFSeed, Ci'))`; `SND(Ci'.Ms'.H(Ms'.PUF(PUFSeed, Ci').T1').T1')`; `local_v` transition 2 verifies the pattern | Captures challenge-response authentication and delivery of the hidden nonce | `Mask(...)` is a symbolic hiding wrapper, not a real XOR implementation |
| Local M3 | `V` recomputes `Ri'`, `Ns'`, `Nd`, `Ci_next`, `SK_local`, `Md`, `Authd`; `ATAj` verifies `Authd` | `local_v` transition 2 computes `Ri_prime'`, `Nd_prime'`, `Ci_next'`, `T2'`, `SK_local'`; `local_ataj` transition 2 reconstructs `Ri_next'`, `Nd_prime`, `SK_local'` | Preserves mutual local authentication and the bounded-session shared-key derivation | Fresh values are symbolic; AVISPA does not model hardware noise or timing |
| Local key update request and acknowledgement | `V -> ATAj : {IDV, tnew, Cnew, Hup}`; `ATAj -> V : {tack, Hack}` | `local_v` transition 3 sends `IDV.Tnew'.Cnew'.Hup'`; `local_ataj` transition 3 verifies the request and sends `Tack'.Hack'`; `local_v` transition 4 verifies the acknowledgement | Preserves authenticated key update and key confirmation | `Tnew'` and `Tack'` are fresh symbolic timestamps, not real time-window checks |
| Cross M1 | `CV -> ATAi : PEnc_PKTA(IDVcv, t3)` | `cross_cv` transition 1 sends `{IDVcv.T3'}_PKTA`; `cross_atai` transition 1 receives the same ciphertext | Only TA can decrypt the vehicle identity and timestamp binding | Public-key encryption is represented symbolically under `PKTA` |
| TA public-key decryption and TA-mediated forwarding | `ATAi -> TA : AEnc_K_TA_Ai(M1, IDAi, t4)`; `TA -> ATAj : AEnc_K_TA_Aj(IDVcv, t5)` | `cross_ta` transition 1 receives `{{IDVcv.T3'}_PKTA.IDAi.T4'}_KTA_Ai`, then sends `{IDVcv.T5'}_KTA_Aj` | Preserves the requirement that TA is the only intermediary between ATAi and ATAj; no direct ATAi <-> ATAj channel is modeled | TA is a symbolic relay and does not learn PUFSeed, raw responses, or vehicle-specific PUF mappings |
| Cross M4 and separate R1/R2 vs P1/P2 branches | `ATAj -> TA : AEnc_K_TA_Aj(C1, C2, P1, P2, t6)` with `R1 = PUF(C1)`, `R2 = PUF(C2)`, `P1 = h(IDVcv, R1, t6)`, `P2 = h(IDVcv, R2, t6)` | `cross_ataj` transition 1 creates `C1'`, `C2'`, `T6'`; computes `R1'`, `R2'`, `P1'`, `P2'`; sends `{C1'.C2'.P1'.P2'.T6'}_KTA_Aj` | Preserves two independent proof branches and prevents collapsing both proofs into a single response branch | Independence is symbolic: two fresh challenges and two separate hash branches |
| Cross M5 | `TA -> ATAi : AEnc_K_TA_Ai(C1, C2, P1, P2, t6, t7)` | `cross_ta` transition 2 forwards `{C1'.C2'.P1'.P2'.T6'.T7'}_KTA_Ai`; `cross_atai` transition 2 receives the same ciphertext | ATAi receives only session-specific proof values and timestamps, not PUFSeed or raw PUF responses | The model assumes protected symmetric forwarding under `KTA_Ai` |
| Cross M6 | `ATAi -> CV : {C1, C2, Authi, t6, t8}` with `Authi = h(P1, t8)` | `cross_atai` transition 2 sends `C1'.C2'.Authi'.T6_store'.T8_store'`; `cross_cv` transition 2 validates the expected hash, derives `P1_prime'`, `P2_prime'`, and `SK_cv'` | Preserves CV-side validation of ATAi and symbolic session-key derivation | The public channel between ATAi and CV allows interception and replay, but not symbolic forgery of the hash preimage |
| Minimal post-verification Authcv confirmation | `CV -> ATAi : Authcv = h(P2, t8)`; `ATAi` accepts only after checking this value | `cross_cv` transition 2 sends `H(P2_prime'.T8')`; `cross_atai` transition 3 accepts only if `H(P2_store.T8_store)` matches | Preserves the final CV-to-ATAi confirmation and the activation of the cross-regional session key | No extra `t9` is introduced; the confirmation is intentionally minimal |
| Symbolic PUF(C) behavior | `PUF(PUFSeed, C)` in both models | `puf(PUFSeed, C)` is abstracted by a private hash-like function in the HLPSL terms | Preserves challenge-dependent deterministic reconstruction shared only by the enrolled vehicle and its home ATA | No physical extraction, cloning, ageing, vibration, or side-channel behavior is modeled |
| Timestamp abstraction | `t0` through `t8`, `tnew`, `tack` | Each timestamp is introduced with `new()` and bound into the corresponding message or hash | Preserves freshness binding and replay resistance at the symbolic level | AVISPA does not check numerical clock windows such as `|t_current - t_i| < Delta t` |
| Session-key derivation | Local: `SK_local = h(Ns, Nd, Ri, Ri_next, t2)` and `SK_new = h(SK_old, Rnew, T, tnew)`; cross: `SK_cv = h(P1, P2, t8)` | `local_v` / `local_ataj` compute `SK_local'` and `SK_new'`; `cross_cv` and `cross_atai` compute `SK_cv'` | Preserves key dependence on fresh transcript material and the PUF-derived branches | Keys remain symbolic messages; AVISPA does not prove computational hardness |
| Local secrecy goal sec_local_key | `secret(SK_local, sec_local_key, {V, ATAj})` | `local_v` and `local_ataj` mark the derived local session key as secret | The attacker should not derive the first local session key | Symbolic secrecy only, under bounded sessions |
| Local secrecy goal sec_updated_key | `secret(SK_new, sec_updated_key, {V, ATAj})` | `local_v` and `local_ataj` mark the updated key as secret | The attacker should not derive the post-update key | Symbolic secrecy only, under bounded sessions |
| Cross secrecy goal sec_cross_key | `secret(SK_cv, sec_cross_key, {CV, ATAi})` | `cross_cv` and `cross_atai` mark the cross-regional session key as secret | The attacker should not derive the cross-regional session key | Symbolic secrecy only, under bounded sessions |
| Cross secrecy goals sec_p1 and sec_p2 | `secret(P1, sec_p1, {CV, ATAi, ATAj, TA})`; `secret(P2, sec_p2, {CV, ATAi, ATAj, TA})` | `cross_ataj` marks the two PUF-derived verification values as secret | The attacker should not learn either proof branch value | The secrecy claim is about the symbolic network attacker, not about a physical probe side-channel |
| Local authentication goals | `local_ataj_auth`, `local_vehicle_auth`, `local_key_update_request`, `local_key_update_ack` | `witness/request` pairs in `local_v` and `local_ataj` on `Ci'.T1'`, `Ci_next'.T2'`, `IDV.Cnew.Tnew`, and `Tack.Tnew` | Preserves vehicle <-> ATAj authentication in the local protocol and the two key-update handshakes | The property is bounded-session authentication, not a full computational proof |
| Cross authentication goals | `cross_atai_auth`, `cross_cv_auth` | `witness/request` pairs in `cross_atai` and `cross_cv` on `P1_prime'.T8'` and `P2_prime'.T8'` | Preserves ATAi authentication by CV and CV authentication by ATAi | The property is tied to the exact symbolic transcript and timestamps used in the HLPSL model |
